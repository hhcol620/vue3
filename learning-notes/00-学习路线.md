# Vue 3 源码学习路线

> 学习策略：从使用场景反推源码，沿主线垂直深入，不横向展开所有细节。

---

## 两条核心主线

```
【启动线】createApp → mount → mountComponent → setupRenderEffect
                                                      ↕ 汇合点
【响应线】ref/reactive → dep.track → dep.trigger → effect.run → 重渲染
```

理解这两条主线的交汇，是读懂 Vue 3 源码的关键。

**为什么在 mountComponent 之后看？**

响应式数据变化 → 重新执行 `componentUpdateFn` → 得到新 vnode → `patch(oldVnode, newVnode)` → diff 决定最小 DOM 操作。理解了完整的组件挂载流程后，diff 的上下文才完整。

## 已学知识地图

```
createApp(App)
  └─► createAppAPI(render)          apiCreateApp.ts
        └─► app.mount('#app')
              └─► render(vnode, container)     renderer.ts
                    └─► patch(null, vnode)
                          └─► processComponent
                                └─► mountComponent
                                      └─► setupRenderEffect
                                            └─► new ReactiveEffect(componentUpdateFn)
                                                  │
                                    ┌─────────────┘
                                    ↓
                            componentUpdateFn
                              └─► renderComponentRoot → vnode
                                    └─► patch(oldVnode, newVnode)

【响应式层】
ref(0) → RefImpl { dep: Dep }
  get .value → dep.track() → Link(activeSub ↔ dep)
  set .value → dep.trigger() → dep.notify() → batch → endBatch → queueJob → 重渲染

reactive(obj) → Proxy
  get → track(target, key)
  set → trigger(target, key)
```
